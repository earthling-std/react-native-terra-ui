# Monorepo migration plan

Convert this `create-react-native-library` (bob) repo from a **library-at-root + nested
`example/` workspace** into a **proper Yarn workspaces monorepo**:

- `src/` → `packages/ui/` (npm package name unchanged: `react-native-terra-ui`)
- `example/` → `apps/example-expo/` (package renamed to `example-expo`)
- root becomes a private orchestrator: `terra-ui-monorepo`

Performed on `main`, using `git mv` to preserve history.

---

## Locked decisions

| Topic | Decision |
|------|----------|
| Package manager | **Yarn 4** (keep `.yarnrc.yml`, `.yarn/`, `nodeLinker: node-modules`, `nmHoistingLimits: workspaces`) |
| UI dir / npm name | dir `packages/ui`; published name stays **`react-native-terra-ui`** @ 0.1.0 |
| Example dir / name | dir `apps/example-expo`; package name **`example-expo`** (`private`, unpublished) |
| Root package | **`terra-ui-monorepo`**, `private: true`, `workspaces: ["packages/*","apps/*"]` |
| TS config | root **`tsconfig.base.json`**; each workspace `tsconfig.json` extends it |
| `#*` subpath imports | **keep**; rely on Metro native `package.json#imports` (package exports); corrected `resolveRequest` only as fallback |
| Turbo | **full pipeline**: `build`, `lint`, `typecheck`, `test`, `build:android`, `build:ios`, `build:web` with `dependsOn`/`outputs`; root scripts fan out via `turbo run` |
| Jest | moves into **`packages/ui`** |
| Biome | single config stays at **root** |
| CI | rewired to `turbo run` + updated paths |
| History / branch | `git mv`, directly on **`main`** |
| Verification | sandbox: `yarn install`, `biome` lint, `tsc` typecheck, bob `build` of `packages/ui`. Native iOS/Android + Metro bundling verified locally by Hai (not possible in sandbox) |

## Execution steps

### 1. Create directories & move files (git mv)
- `mkdir -p packages apps`
- `git mv src packages/ui/src`
- `git mv tsconfig.build.json packages/ui/tsconfig.build.json`
- `git mv example apps/example-expo`
- Move (then edit) `babel.config.js` → `packages/ui/babel.config.js` (root library babel; the example has its own).

### 2. Root `package.json` → `terra-ui-monorepo`
- Set `"name": "terra-ui-monorepo"`, `"private": true`.
- **Remove** library fields: `version`, `main`, `types`, `imports`, `exports`, `files`,
  `keywords`, `publishConfig`, `react-native-builder-bob`, `jest`, `create-react-native-library`,
  `peerDependencies`, library `repository/author/license/bugs/homepage` (keep repo-level ones if desired).
- `"workspaces": ["packages/*", "apps/*"]`.
- Keep `packageManager: yarn@4.11.0`.
- Root `devDependencies`: keep only orchestration-level (`turbo`, `@biomejs/biome`, `typescript`); push RN/bob/jest/react devDeps down to `packages/ui`.
- Scripts (fan out via turbo):
  - `"lint": "turbo run lint"`, `"lint:fix": "biome check --write ."`,
    `"format": "biome format --write ."`,
    `"typecheck": "turbo run typecheck"`, `"test": "turbo run test"`,
    `"build": "turbo run build"`,
    `"example": "yarn workspace example-expo"`.

### 3. `packages/ui/package.json` (the library)
- New file carrying everything stripped from old root: `name: react-native-terra-ui`,
  `version`, `main`, `types`, `imports: { "#*": "./src/*" }`, `exports` (`.`, `./theme`, `./package.json`),
  `files`, `keywords`, `publishConfig`, `react-native-builder-bob`, `peerDependencies`,
  `repository/homepage/bugs/author/license`.
- `devDependencies`: bob, jest, @react-native/*, react, react-native, unistyles, types, del-cli,
  babel-plugin-module-resolver, etc.
- **Jest** block moved here; `modulePathIgnorePatterns` → `["<rootDir>/lib/"]` (example no longer nested).
- Scripts: `"build": "bob build"` (replaces old `prepare`), `"clean": "del-cli lib"`,
  `"typecheck": "tsc"`, `"lint": "biome check ."`, `"test": "jest"`.
  Keep a `prepare`/`prepack` → `bob build` for publish correctness.

### 4. TS configs
- New root **`tsconfig.base.json`**: all strict options from current root tsconfig
  (`strict`, `verbatimModuleSyntax`, `customConditions`, `moduleResolution: bundler`, etc.),
  **minus** `rootDir` and `paths`.
- `packages/ui/tsconfig.json`: `extends ../../tsconfig.base.json`; adds
  `paths: { "react-native-terra-ui": ["./src/index"], "react-native-terra-ui/theme": ["./src/theme/index"], "#*": ["./src/*"] }`, `rootDir: "."`.
- `packages/ui/tsconfig.build.json`: `extends ./tsconfig`; `exclude: ["lib"]` (no more `example`).
- `apps/example-expo/tsconfig.json`: `extends ../../tsconfig.base.json` (Expo-compatible).

### 5. `packages/ui/babel.config.js`
- Move existing root babel here; `module-resolver` alias `^#(.+) → ./src/\1` stays correct
  (package-relative). No path change needed beyond location.

### 6. `apps/example-expo` wiring
- `package.json`: `"name": "example-expo"`; keep `"react-native-terra-ui": "workspace:*"`.
- `metro.config.js`: `root = path.resolve(__dirname, '../..')` (monorepo root, was `'..'`).
  **First attempt:** enable Metro package-exports (`config.resolver.unstable_enablePackageExports = true`)
  and **delete** the `#`-rewriting `resolveRequest` hack (Metro honors `package.json#imports`).
  **Fallback** if Metro can't resolve `#…` against the linked source: keep a `resolveRequest` that
  resolves `#` against the **ui package** dir (`packages/ui/src`), not the monorepo root.
- `babel.config.js`: `root = path.resolve(__dirname, '../..')`; `pkg = require('../../packages/ui/package.json')`
  (was `'../package.json'`). Unistyles plugin `autoProcessImports: ['react-native-terra-ui']` unchanged.
- `index.js`, `app.json`, `src/`, `assets/` unchanged.

### 7. `turbo.json` — full pipeline
- Define tasks:
  - `build`: `outputs: ["lib/**"]`, library-scoped.
  - `typecheck`, `lint`, `test`: `outputs: []`.
  - `build:android`: `dependsOn: ["^build"]`, env unchanged, `inputs` repointed
    (`packages/ui/src/**`, `apps/example-expo/package.json`, `apps/example-expo/android`, ignores).
  - `build:ios`: same treatment (`apps/example-expo/ios`, podspec lives in `packages/ui` now → adjust input).
  - `build:web`: example-scoped.
- example `build:android/ios` `dependsOn: ["react-native-terra-ui#build"]` so the lib builds first.

### 8. `.gitignore`
- Repoint example native ignores: `example/ios` → `apps/example-expo/ios`,
  `example/android` → `apps/example-expo/android`, `example/vendor/` → `apps/example-expo/vendor/`.
- `lib/` ignore still matches `packages/ui/lib/` (relative, fine), but make explicit if needed.

### 9. CI (`.github/workflows/ci.yml`)
- lint/typecheck/test jobs → `yarn turbo run lint` / `typecheck` / `test`.
- `build-library` → `yarn turbo run build --filter=react-native-terra-ui` (was `yarn prepare`).
- `build-android` / `build-ios` → keep `yarn turbo run build:android` / `build:ios`;
  Gradle cache key path `example/android/gradle/...` → `apps/example-expo/android/gradle/...`;
  prebuild `yarn example expo prebuild ...` → `yarn workspace example-expo expo prebuild ...`.
- `build-web` → `yarn workspace example-expo build:web`.
- Setup action + Yarn cache (`**/node_modules`) unchanged.

### 10. Verify (sandbox)
- `yarn install` (regenerate workspace links / lockfile).
- `yarn turbo run lint typecheck` — Biome + tsc clean across workspaces.
- `yarn turbo run build --filter=react-native-terra-ui` — bob emits `packages/ui/lib`.
- `yarn turbo run test` — jest passes in `packages/ui`.
- Confirm `git status` shows renames (history preserved).

### 11. Local follow-up (Hai)
- `yarn workspace example-expo start` — Metro resolves `react-native-terra-ui` + `#` imports.
- `yarn workspace example-expo ios` / `android` (prebuild) — native smoke test.
- If Metro fails on `#…`, apply the §6 fallback resolver.

## Risks / watch-points
- **Metro package-exports** (`#` resolution) is the main unknown — fallback resolver documented.
- **bob `source/output`** are package-relative (`src`/`lib`) → no change needed once in `packages/ui`.
- **`*.podspec` / native input paths** in `turbo.json build:ios` move with the library to `packages/ui`.
- **Yarn `nmHoistingLimits: workspaces`** keeps RN deps un-hoisted per workspace — preserved.
- No committed native binaries (gitignored) → move is light; native dirs regenerate via prebuild.
